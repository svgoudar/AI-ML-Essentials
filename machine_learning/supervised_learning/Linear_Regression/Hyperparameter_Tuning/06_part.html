Cross-Validation#

Why do we need Cross-Validation#

  • When training ML models, we split data into:

    • Training set ‚Üí model learns patterns.

    • Validation set ‚Üí used for hyperparameter tuning.

    • Test set ‚Üí unseen data, used only at the end to check performance.

If we simply split data once (say 70% train, 30% test), the result depends on the random split. üëâ Cross-validation helps us get a more reliable performance estimate by training and validating on multiple splits.

Types of Cross-Validation#

1. Leave-One-Out CV (LOOCV)#

  • Take one data point as validation, rest as training.

  • Repeat for every data point.

  • Accuracy = average of all experiments.

‚úÖ Advantage: maximum use of training data. ‚ùå Disadvantages:

  • Computationally expensive (n experiments for n records).

  • Prone to overfitting (since validation set = 1 record).

2. Leave-P-Out CV#

  • Instead of leaving 1 record, leave p records as validation.

  • Train on the rest.

  • Repeat for all possible combinations.

‚úÖ More flexible than LOOCV. ‚ùå Impractical for large datasets (combinatorial explosion).

3. K-Fold Cross-Validation#

  • Split dataset into k equal folds.

  • Train on k-1 folds, validate on the remaining fold.

  • Repeat k times (each fold used once as validation).

  • Final score = average of all folds.

‚úÖ Balance between efficiency and reliability. ‚úÖ Most common method in ML. ‚ùå Validation set may not preserve class distribution (in classification).

4. Stratified K-Fold CV#

  • Same as K-Fold, but ensures class distribution is preserved in each fold.

  • Example: if dataset has 60% positive and 40% negative labels ‚Üí each fold keeps ~60:40 ratio.

‚úÖ Very important for classification tasks with imbalanced data.

5. Time Series Cross-Validation#

  • In time series, order matters ‚Üí can‚Äôt randomly shuffle.

  • Train on past data, validate on future data.

  • Example:

    • Train = Day 1‚Äì100, Validate = Day 101‚Äì120

    • Train = Day 1‚Äì120, Validate = Day 121‚Äì140

‚úÖ Used in forecasting, stock prediction, sentiment analysis. ‚ùå Training size keeps growing ‚Üí computationally heavier.

üîπ Why Cross-Validation?#

  • Provides stable estimate of performance.

  • Helps avoid overfitting by testing model on multiple validation sets.

  • Essential for hyperparameter tuning (GridSearchCV, RandomizedSearchCV, Bayesian Optimization).
